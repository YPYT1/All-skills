# Unified Skills Repository - 使用指南

## 概述

这是一个完整的解决方案，帮助你整合多个 GitHub 仓库中的 AI Agent Skills，并实现自动同步更新。

## 核心特性

✅ **自动同步** - GitHub Actions 每天自动检查并更新所有 skills  
✅ **统一管理** - 所有 skills 集中在一个仓库中管理  
✅ **保留历史** - 使用 Git Submodules 保留原始仓库的完整历史  
✅ **灵活配置** - 支持不同结构的仓库  
✅ **易于扩展** - 轻松添加新的 skills 来源

## 快速开始

### 1. 解压文件

```bash
tar -xzvf unified-skills-repo.tar.gz
cd unified-skills-repo
```

### 2. 初始化仓库

```bash
# 初始化 Git 仓库并添加所有子模块
./setup.sh
```

这个脚本会：
- 初始化 Git 仓库
- 添加所有 skills 来源作为子模块
- 下载所有 skills 内容
- 配置同步脚本

### 3. 推送到 GitHub

```bash
# 创建 GitHub 仓库（在 GitHub 网站上操作）
# 然后推送代码
git remote add origin https://github.com/yourusername/unified-skills.git
git push -u origin main
```

### 4. 启用自动同步

在 GitHub 仓库中：
1. 进入 **Settings > Actions > General**
2. 确保 Actions 已启用
3. 自动同步将每天 UTC 00:00 运行

## 日常使用

### 更新所有 Skills

```bash
# 更新到最新版本
./scripts/update-all.sh

# 更新并自动提交
./scripts/update-all.sh --commit
```

### 同步到统一目录

```bash
# 同步所有来源到 skills-export/ 目录
./scripts/sync-skills.sh

# 同步特定来源
./scripts/sync-skills.sh anthropics
```

### 在 Claude Code 中使用

```bash
# 方式 1: 直接安装单个 skill
claude skills install ./skills/anthropics/skills/algorithmic-art

# 方式 2: 复制所有 skills 到 Claude Code 目录
cp -r skills/anthropics/skills/* ~/.claude/skills/

# 方式 3: 使用同步后的目录
cp -r skills-export/anthropics/* ~/.claude/skills/
```

### 在 Cursor 中使用

```bash
mkdir -p ~/.cursor/skills
cp -r skills/anthropics/skills/* ~/.cursor/skills/
```

## 添加新的 Skills 来源

### 步骤 1: 添加子模块

```bash
git submodule add https://github.com/username/new-skills.git skills/new-source
```

### 步骤 2: 更新配置

编辑 `scripts/sync-config.json`，添加新来源的配置：

```json
{
  "name": "new-source",
  "displayName": "New Skills Source",
  "repo": "https://github.com/username/new-skills",
  "localPath": "skills/new-source",
  "skillPaths": ["skills"],
  "exclude": ["docs", "tests"],
  "description": "简短描述"
}
```

### 步骤 3: 提交更改

```bash
git add .
git commit -m "feat: add new skills source from username/new-skills"
git push
```

## 自动同步配置

### 修改同步频率

编辑 `.github/workflows/sync-skills.yml`：

```yaml
on:
  schedule:
    # 每天 UTC 00:00（默认）
    - cron: '0 0 * * *'
    
    # 每小时
    # - cron: '0 * * * *'
    
    # 每周一
    # - cron: '0 0 * * 1'
```

### 配置同步行为

在 GitHub Actions 中设置环境变量：

```yaml
env:
  AUTO_COMMIT: "false"    # 是否自动提交
  CREATE_PR: "true"       # 是否创建 PR
```

## 故障排除

### 问题 1: 子模块为空

```bash
# 解决方案
./scripts/init-submodules.sh
# 或
git submodule update --init --recursive
```

### 问题 2: 更新失败

```bash
# 强制更新
git submodule update --remote --force

# 重新初始化
rm -rf .git/modules/skills/*
git submodule update --init
```

### 问题 3: 权限错误

确保 GitHub Token 有足够权限：
- `contents: write` - 写入仓库内容
- `pull-requests: write` - 创建 PR

## 工作原理

### Git Submodules

Git Submodules 允许你将一个 Git 仓库作为另一个仓库的子目录：

```
主仓库
├── .gitmodules (配置)
└── skills/
    ├── anthropics/ → https://github.com/anthropics/skills
    ├── superpowers/ → https://github.com/obra/superpowers
    └── ...
```

### 自动同步

GitHub Actions 每天执行：
1. 检出仓库（包含所有子模块）
2. 运行 `git submodule update --remote` 更新到最新
3. 创建 Pull Request 或直接提交

### 目录映射

不同仓库的 skills 位置不同：
- `anthropics/skills` → `skills/` 子目录
- `obra/superpowers` → `skills/` 子目录
- `sickn33/antigravity` → 根目录

同步脚本会根据配置自动处理这些差异。
